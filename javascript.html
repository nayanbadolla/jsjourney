    • Namaste JavaScript

    Intro

  • JS is single threaded synchronous language
  • Execution Context consists of 2 components
    - Variable Component (Memory Environment)
    - Code Component (Thread of Execution)

    • Execution Context | Call Stack

  • parameter is also assigned memory in Memory Phase of Execution Context
  • Call Stack maintains order of execution of execution contexts
  • Called=Execution Context/Program/Control/Runtime/Machine Stack

    • Hoisting

  • based on execution context and its phases
  • different way of working for different syntaxes
  • existing before initialization

    • Functions

  • restricted scope

    Scope Chain

  • lexical environment
  • parent execution context

    • let vs const | Temporal Dead Zone | errors types

  • let, const stored in Script and not in Global
  • let can be initialized later; const cannot be initialized later
  • temporal dead zone - phase between hoisting and initialization; present for let and const
  • reference error - accessing not defined variable or variable in temporal dead zone
  • syntax error - not initializing while declaring const, redeclaring variable
  • type error - changing const type

    • Block Scope | Shadowing

  • let, const stored in Block {} when present
  • each Block scope is separate, follows lexical environment
  • shadowing is having different scope for same variable name; exists for let and const type
  • illegal shadowing is declaring let in Script and var in Block

    • Closure

  • presence of function within a function where inner function exists in outer function with its lexical environment
